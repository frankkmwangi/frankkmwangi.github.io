Password Policy Enforcement

Problem Statement

Many organizations struggle with weak password practices that leave systems vulnerable to brute-force and dictionary attacks. This project focused on enforcing strong password policies in a Windows environment.

Approach

  • Used the Local Security Policy editor to define custom password rules.
  • Configured settings such as minimum length, complexity requirements, and expiration period.
  • Simulated user password creation attempts to validate the policy.

Tools Used

  • Windows 10 Local Security Policy (secpol.msc)
  • Group Policy Editor (for domain environments)
