Network Packet Analysis using Wireshark

Problem Statement

Analyze suspicious network traffic to identify potential security breaches or abnormal behavior within a LAN environment.

Approach

  • Used a publicly available Wireshark capture file (PCAP) to simulate packet analysis.
  • Filtered traffic by IP, protocol (HTTP, DNS), and TCP flags.
  • Reviewed suspicious packets for signs of data exfiltration or port scanning.

Tools Used

  • Wireshark (open source)
  • Sample PCAP file from Wireshark Sample Captures

Key Lessons Learned

  • How to use filters like http, tcp.flags.syn == 1, and ip.addr == x.x.x.x.
  • Recognizing patterns of scanning and large data transfers.
  • Importance of traffic inspection in identifying cybersecurity threats.
